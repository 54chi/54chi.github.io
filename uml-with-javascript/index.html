UML with Javascript

19 February 2016 on blog

As my projects are getting more and more complex, I'm going to need a better way to describe them.

Enter UML.

Typically, an Architect or Designer will use something like Visio and then convert the diagram into an image before sharing it, assuming such Architect is stuck in the 1990's (great decade, but is time to let it go)

A better way for me is to use a semantic way to describe my diagrams (aka: text), so is easier to update and works very well with version control. There are many solutions out there, but I set up for mermaid. It runs straight from the browser and the format is very similar to PlantUML, another favorite. It doesn't have all the diagrams in the universe, but for what I need is more than enough.

Here is one example:
